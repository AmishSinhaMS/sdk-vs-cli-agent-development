# Contributing

Thanks for your interest in improving this content! Contributions, corrections, and new perspectives are welcome.

## What We're Looking For

- **Factual corrections** — if a framework description, link, or capability is outdated or inaccurate
- **Additional perspectives** — real-world experiences with SDK vs CLI agent development
- **New scenarios** — use cases or patterns we haven't covered
- **Framework updates** — new tools, SDKs, or CLI agents that should be included
- **Clarity improvements** — better explanations, diagrams, or examples

## How to Contribute

1. **Fork** this repository
2. **Create a branch** for your changes (`git checkout -b improve-sdk-section`)
3. **Make your edits** to the blog post in `blog/sdk-vs-cli-agent-development.md`
4. **Submit a Pull Request** with a clear description of what you changed and why

## Guidelines

- Keep the tone **neutral and developer-focused** — we're comparing tools, not picking sides
- **Cite sources** when adding claims about specific frameworks or capabilities
- Maintain the **existing structure** (headings, tables, code blocks) for consistency
- Keep the post **practical** — focus on "when to use what" rather than theoretical arguments

## Reporting Issues

If you spot an error but don't want to submit a PR, [open an issue](../../issues/new) with:
- What's incorrect
- What it should say (if you know)
- A link or source for the correction

## Code of Conduct

Be respectful, constructive, and assume good intent. We're all here to learn and help each other build better software.
